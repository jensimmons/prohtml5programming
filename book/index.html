Possibility Pros Cons Example Verdict
Turn.js-based website Looks like a book. Uses a page turning metaphor familiar to users of digital book readers. Because the book exists as a website, it's easy to setup free access to a sample and easy to link to external websites. Laborious and tedious to place content. Text doesn't resize. Fixed-sized book doesn't resize for different size screens. Requires infrastructure for a paywall restricting and managing access a website. Chapter 2  
Responsive-web-design-based website Works on a wide-range of devices and sizes of screen. Fully accessible, text resizes. Because the book exists as a website, it's easy to setup free access to a sample and easy to link to external websites. Might not "look like a book" with sexy page turning animations. Might feel like a website. (TBA, design exploration needed to see what we might come up with.) Requires infrastructure for a paywall restricting and managing access a website.    
iBooks Author book Provides a book metaphor in a popular and powerful app. Authoring tool makes it easy to place long-form content. Might allow us to use HTML5 inside a book (further exploration needed). Apple provides ecommerce platform for books with marketing tools. The product makes sense as a thing a customer has bought — a file to download and keep. Book product only works in iBooks (iPhone/iPad/iPod only). Can only be sold through the iBooks store. Apple gets a 30% cut.   Need to just try it out, and create an iBooks Author book examples to learn more.
iOS Mobile App (HTML5 wrapped up as an app via PhoneGap) Provides all the technology we need. The product makes sense as a thing a customer has bought — a file to download and keep. While a lot of publishers are making books into apps, books and apps are different creatures in most people's minds. Sold through the Apple App store. Apple gets a 30% cut.    
Traditional eBook files: ePub, Mobi, DAISY and PDF Known quantity. One file format or another will work on all ebook readers. Can place videos in some file formats. Can't implement any HTML5 technology. No possibilities for the reader to edit code examples. No live demonstrations. Provides nothing new.   No
EPUB 3 book Might be awesome. Might be too early. Existing ebook readers don't support the format yet.   More research needed
